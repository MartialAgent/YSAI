## QAG 작동 방식

```
태스크 프롬프트 → GPT가 3개 퀴즈 질문 + 예상 답변 생성
워커 응답 → GPT가 응답만 보고 각 질문에 답
결과: D = 1 - (정답 수 / 전체)   →   D=0.000 = 완벽, D=1.000 = 완전 실패
```

**기준선 대비**: 기준선 D는 단일 주관적 점수. QAG는 질문별 감사 가능한 분석 제공.

## ✅ 잘 된 점

- **E_ChurnFormula 누락 변수 감지 (D=0.333)**: QAG가 4개 필수 변수에 대한 질문을 생성하고 `support_tier_weight` 답변이 실패. 의도된 누락 감지가 정확히 작동.
- **C_AdCopy D=0.333 (기준선 ~0.037)**: QAG가 광고 카피의 정확한 ROI% 미인용을 포착. 기준선 비평가 대비 약 9배 민감도 향상.
- **낮고 안정적인 A-Score**: 최대 A-Score 0.062 — Exp03(0.204)보다 훨씬 안정적. QAG가 더 깔끔한 편차 신호 생성.
- **질문별 감사 가능한 분석**: 종합 비평가 점수와 달리 어떤 질문이 왜 실패했는지 정확히 파악 가능.

## ❌ 부족한 점

- **D_ConditionAudit QAG 맹점**: 퀴즈가 감사된 조건 1–4에 대해 묻고 모두 통과. "조건 5번이 점검되었는가?"를 묻지 않음 — 태스크 프롬프트에서 생성된 것이지 정답 체크리스트에서 나온 것이 아니기 때문. **QAG는 말한 것을 검증하지, 말하지 않은 것을 검증하지 않음.**
- **F_BoardReport가 불완전한 상위 데이터에도 3/3 통과**: F가 누락 변수가 있는 E의 데이터를 인용하지만 퀴즈는 통과. QAG가 상위 누락 지식을 전파하지 못함.
- **퀴즈 품질이 GPT 질문 생성에 의존**: GPT가 "올바른" 질문을 생성하지 못하면 실패를 놓침. 구조적 약점 — 퀴즈 출제자와 워커가 동일 모델.
