## 수식 (기준선 대비 변경 사항)

### 기존 수식 (Exp01–02 기준선):
```
A = α(1 - S) + β × D × (1 - C)

S = 응답 2개 간 코사인 유사도
C = 워커 자기보고 신뢰도 (항상 ~0.8–1.0)
D = GPT 비평가 단일 점수
```

### 새 수식 (Exp08):
```
A = α(1 - W) + β × (D / C_adj) × U

W     = Kendall's W 프록시 — k=3 응답 간 평균 쌍별 코사인 유사도
D     = 불일치도 — 주 응답이 그룹 평균에서 벗어나는 정도
Z     = 주 응답의 우려/불확실성 키워드 수
C_adj = C × (1 / (1 + ln(1 + Z)))   ← 표현된 우려로 신뢰도 감쇠
U     = 긴급도 승수 (일반 QA는 1.0)
```

### 각 항목의 역할:

| 구성요소 | 역할 | 중요한 이유 |
|---------|------|-----------|
| **W (Kendall's)** | 3개 응답 간 그룹 일치도 측정 | 2응답 코사인 S보다 강건함; 2/3이 동의하지만 1개가 다를 때 포착 |
| **D (불일치도)** | 주 응답 vs. 그룹 편차 | 특정 응답이 이상치일 때 식별 |
| **C_adj** | 우려 키워드로 조정된 신뢰도 | 텍스트에서 표현된 불확실성을 높은 자기보고 C가 가리는 것 방지 |
| **U** | 긴급도 승수 | 도메인별 증폭 허용 (의료, 금융) — 여기서는 1.0 |

## ✅ 잘 된 점

- **C_adj가 핵심 변별자**: F_BoardReport와 G_RiskAudit 둘 다 Z=4 (불완전 데이터 합성 시 모델이 헷지 표현 많이 사용). C_adj가 0.345로 떨어져 D/C_adj를 크게 증폭. 이것이 의도된 동작 — 모델 자체가 불확실성 신호를 보낼 때 수식이 더 엄격하게 작동함.

- **E_ChurnFormula Z=2**: 의도적으로 불완전한 이탈 수식(`support_tier_weight` 누락) 생성 시 모델이 두 번 헷지. C_adj가 0.900에서 0.429로 하락 — 외부 비평가 없이도 우려 감쇠가 누락 변수에 대한 모델 내부 불확실성을 올바르게 포착.

- **D_ConditionAudit A=0.028 (실험 최고 점수)**: 3개 응답 모두 거의 동일한 감사 보고서 생성 (W=0.971, D=0.028), Z=0 (헷징 없음). 감사 노드가 확신 있고 일관적 — C_adj=1.000 유지. 이 안정적 노드에 대해 가장 낮은 A-Score가 정확함.

- **A_Pricing D=0.077 (가장 높은 원시 불일치도)**: 3개 가격 응답 간 티어 명명과 구조 변동이 더 많음. 비평가 호출 없이도 수식이 올바르게 포착.

- **위양성 없음**: 깨끗한 노드 (B_ROI, D_ConditionAudit, C_AdCopy) 모두 0.070 이하 유지.

## ❌ 부족한 점

- **외부 비평가 없음 (D는 자기 측정)**: D는 워커 응답 간 임베딩 거리로 계산됨, 비평가의 정확성 평가가 아님. 응답이 일관되게 틀릴 수 있음 (낮은 D, 높은 W)에도 낮은 A-Score를 받을 수 있음. E_ChurnFormula: D=0.034 (안정)인데 의도적 변수 누락 — 수식이 내용 오류가 아닌 응답 간 분산만 포착.

- **Z 키워드 카운팅이 조악함**: "however", "note that", "assumption"이 맥락과 관계없이 동등하게 처리됨. "Note that this is a best-practice approach"라는 응답은 내용이 완벽히 정확해도 Z=1을 받음. 잘 보정된 신중한 출력을 과도하게 불이익 줄 수 있음.

- **U=1.0 항상**: 긴급도 승수가 이 실험에서 사용되지 않음. 고위험 도메인 (의료, 금융)에서 U를 높여야 하지만 노드 또는 도메인별로 설정하는 메커니즘 아직 없음.

- **C_adj=0.345인 F/G가 AMBIGUOUS 유발 안 함**: A=0.156과 A=0.113이 0.4 임계값 이하. 높은 우려 카운트와 감쇠된 신뢰도에도 불구하고 시스템이 이 노드들을 통과시킴. 도메인별 임계값 또는 C_adj 하한 트리거가 필요함.

## 🔧 다음에 필요한 것

- **외부 D 신호 추가**: C_adj 감쇠를 QAG 퀴즈 질문 또는 G-Eval 루브릭 점수와 결합 — 내부 불확실성(Z)과 외부 정확성 확인 둘 다 확보
- **도메인별 U 값**: 노드 또는 그래프 정의에서 U 지정 허용 (예: 금융 노드 U=2.0)
- **C_adj 임계값 라우팅**: C_adj < 0.4이면 A-Score와 관계없이 AMBIGUOUS로 라우팅 — 신뢰하기에 너무 많은 불확실성 표현
- **더 스마트한 Z 카운팅**: 맥락별 우려 키워드 가중치 (예: 창의적 브리프의 "assumption"은 무해; 의료 진단의 "assumption"은 심각)
