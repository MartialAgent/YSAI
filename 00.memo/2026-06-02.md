## [2026-06-02 14:39] python_syntax — 언패킹 문법 (`*`, `**`)

**핵심**: 컨테이너를 "통째로 1개"가 아니라 **풀어서 여러 개**의 인자로 전달.

- `*` → 리스트/튜플을 **위치 인자**로 풀기 (순서대로 매칭)
- `**` → 딕셔너리를 **키워드 인자**로 풀기 (키 이름으로 매칭, 순서 무관)

```python
def greet(name, age): ...
info = {'name': '나연', 'age': 25}

greet(info)     # dict 1개 → 에러
greet(**info)   # greet(name='나연', age=25) 로 풀려서 호출 ✓
```

**오해 정리**:
- `User(**user_data)` 는 타입 힌팅이 **아님** → 그냥 **객체 생성**.
- 딕셔너리를 키워드 인자로 풀어 `User.__init__(id=..., name=..., email=...)` 호출하는 것.
- Pydantic `BaseModel` 은 이 생성 시점에 **런타임 타입 검증** 수행 → 틀리면 `ValidationError`.
- `TypedDict` 는 검증 없음 → 잘못된 타입도 그냥 통과.

**기타 자주 보는 언패킹**:
- `a, *rest = [1,2,3,4]` (변수 할당)
- `def f(*args, **kwargs)` (가변 인자)
- `[*a, *b]`, `{**d1, **d2}` (컨테이너 합치기)

## [2026-06-02 15:24] python_syntax — `*args` 모으기 & dict 합치기 실습

**가장 큰 깨달음**: `*` 는 위치에 따라 **방향이 정반대**다.

| 위치 | `*` 의미 |
|------|---------|
| **호출** 시: `f(*nums)` | 컨테이너를 **풀어서** 인자로 |
| **정의** 시: `def f(*nums)` | 인자들을 **모아서** 튜플로 |

같은 기호인데 풀기 vs 모으기 — 가장 헷갈리는 포인트.

**`*args` 모으기 확인**:
```python
def show(*nums):
    print(nums)

show(10, 20, 30)   # 출력: (10, 20, 30)   ← 튜플로 모임 (괄호 포함)
show('a', 'b')     # 출력: ('a', 'b')      ← 문자열은 따옴표 함께 출력
```

**내가 막힌 지점**: `{**d1, **d2}` 결과를 `(1, 2)` 라고 답함.
→ `**` 가 **값만** 풀어내는 줄 알았음. 사실은 **키:값 쌍 통째로** 풀어냄.
→ "여기서는 키값쌍이 된다는 걸 알려준 건가" 라고 깨달음.

핵심 규칙:
- dict는 **`키: 값` 이 1개의 단위** → 풀어도 키가 따라옴
- 바깥 괄호가 `{}` 면 결과도 dict
- `{1, 2}` 는 **set** (값만 있음), `{'a': 1}` 는 dict — 다른 자료형

**최종 작성한 함수** (merge_dicts 빈칸 채우기로 완성):
```python
def merge_dicts(*dicts):       # * 로 여러 dict 를 튜플로 모음
    result = {}
    for d in dicts:
        result.update(d)        # 같은 키는 덮어씀
    return result

merge_dicts({'a':1}, {'b':2}, {'a':99, 'c':3})
# → {'a': 99, 'b': 2, 'c': 3}
```

**한 줄 버전** (메모용, 나중에 익혀볼 것):
```python
def merge_dicts(*dicts):
    return {k: v for d in dicts for k, v in d.items()}
```

**규칙 한 줄**: `*` 뒤 변수 이름은 본문에서 사용하는 이름과 **같아야** 한다 (`*dicts` 로 받았으면 본문에서도 `dicts`).

## [2026-06-02 15:35] python_syntax — `combine_lists` 연습 (같은 패턴 정착)

**같은 골격을 dict → list 로 옮긴 연습**. 패턴 머슬 메모리화.

```python
def combine_lists(*lists):
    result = []
    for lst in lists:
        result += lst
    return result

combine_lists([1, 2], [3, 4, 5], [6])  # → [1, 2, 3, 4, 5, 6]
```

**또 막힌 지점들** (반복 실수):
1. `*lst` 라고 적음 → 본문은 `lists` 인데. **`*` 뒤 이름이 본문과 안 맞음** (이번이 두 번째).
2. 빈 컨테이너로 `{}` 적음 → 그건 dict. **list 의 빈 표기는 `[]`**.
3. `for lst in lists:` 보고 "lst로 적혀있는데 list가 맞니?" 라고 질문.
   → **`lst` (loop 변수, 임시) 와 `lists` (collection, 시그니처 변수) 는 서로 다른 두 변수**.
   → 비유: `for 사과 in 사과상자:` — 상자 이름과 사과 이름은 다름.

**빈 컨테이너 표기 표** (외울 것):
| 자료형 | 빈 표기 |
|--------|---------|
| list | `[]` |
| dict | `{}` |
| tuple | `()` |
| set | `set()` |

**merge_dicts ↔ combine_lists 골격 비교**:
| | merge_dicts | combine_lists |
|---|------------|---------------|
| 빈 시작값 | `{}` | `[]` |
| 합치는 방법 | `.update(d)` | `+= lst` |

→ 골격이 같으면 **자료형만 바꿔서 응용 가능**. 다음엔 set, tuple, str 등으로 확장 연습.

## [2026-06-02 15:46] python_syntax — `**kwargs` 도입 & 오늘의 결론

**`**kwargs` 처음 도입** — 버그 찾기 문제로 자연스럽게.

```python
def make_record(**fields):     # 키워드 인자들을 fields 라는 dict 로 모음
    result = {}
    for key in fields:
        result[key] = fields[key]
    return result

make_record(name='A', age=20)
# → {'name': 'A', 'age': 20}
```

**막혔던 지점**: `name='A', age=20` 을 "튜플"이라고 부름.
→ 사실은 **키워드 인자**라는 별도 개념. 튜플도 dict도 아님.
→ 호출 형태별 분류:
| 호출 | 이름 |
|------|------|
| `f(1, 2, 3)` | 위치 인자 |
| `f(name='A', age=20)` | **키워드 인자** |
| `f((1,2,3))` | 튜플 1개 (위치 인자) |
| `f({'a':1})` | dict 1개 (위치 인자) |

---

### 🎯 오늘의 결론 (본인 표현 그대로)

> **"정의할 때는 모으고, 호출할 때는 푸는 거고, `*` 는 리스트/튜플, `**` 는 딕셔너리로 타입이 다른 거네"**

이 한 문장이 `*`, `**` 의 전부.

---

**최종 정리표** (외울 것):

| 시그니처 | 호출 | 함수 안 변수 |
|---------|------|------------|
| `def f(x):` | `f(value)` | `x = value` |
| `def f(*args):` | `f(v1, v2, v3)` | `args = (v1, v2, v3)` **튜플** |
| `def f(**kwargs):` | `f(k1=v1, k2=v2)` | `kwargs = {'k1': v1, 'k2': v2}` **dict** |

**별표 1개는 list/tuple 쪽, 별표 2개는 dict 쪽** — 항상 같이 다님.

**모으기 결과는 고정 자료형** (`*args` → 튜플, `**kwargs` → dict).
**풀기 입력은 유연** (`f(*x)` 는 순회 가능한 거면 뭐든).

## [2026-06-02 16:10] python_syntax — 매개변수/인자 용어 정리 (가장 중요한 기초)

**나는 Martial Agent** — 에이전트 배우기 위해 치열하게 싸우는 싸움꾼.

오늘 호출/정의를 헷갈리는 근본 원인: **용어 자체를 안 잡고 있었음**. 여기서 잡고 간다.

### 핵심 표 — 시점에 따라 부르는 이름이 다르다

| 한국어 | 영어 | 어디서? | 무엇? |
|--------|------|--------|------|
| **매개변수** | parameter | 함수 **정의** 시 | 받을 자리 (이름) |
| **인자** | argument | 함수 **호출** 시 | 실제 값 |

```python
def log(msg, level='INFO'):     # msg, level = 매개변수
    ...
log('hello', level='WARN')      # 'hello', 'WARN' = 인자
```

### "위치/키워드" vs "필수/선택" 구분

```python
# 정의 시: 필수 vs 선택 (기본값 유무로 결정)
def log(msg, level='INFO'):
#       │    │
#       필수  선택 (기본값 'INFO' 있음)

# 호출 시: 위치 vs 키워드 (표기 형태로 결정)
log('hello')                # 'hello' = 위치 인자
log('hello', 'WARN')        # 둘 다 위치
log('hello', level='WARN')  # 'hello'=위치, level='WARN'=키워드
log(msg='hi', level='W')    # 둘 다 키워드
```

**핵심 함정**: "level='INFO'" 처럼 기본값 있다고 해서 **반드시 키워드로만 부르는 건 아님**. 위치로도 호출 가능. 정의의 "선택 매개변수" ≠ 호출의 "키워드 인자".

### 본인 잘못 알고 있던 표현 (교정 기록)

> ❌ "msg라는 **위치인자**와 level이라는 **키워드 인자**를 받아서"

✓ 교정:
> "msg는 **필수 매개변수**, level은 **기본값이 있는 선택 매개변수**"

### `*args`, `**kwargs` 의 진짜 의미 (이제 이해됨)

- `args` = **arg**ument**s** 의 약자 (인자들)
- `kwargs` = **k**ey**w**ord **arg**ument**s** (키워드 인자들)
- `*args` → 위치 **인자**들을 모으는 **매개변수**
- `**kwargs` → 키워드 **인자**들을 모으는 **매개변수**

→ 이름 자체가 "인자들을 모은다" 는 뜻을 그대로 담음.

### 분류 연습 (정답)

```python
greet('hello', 5, name='Martial', loud=True)
       └─────┘ └┘  └────────────┘ └───────┘
       위치    위치  키워드          키워드
```

기준: 값만 있으면 위치, `이름=값` 형태면 키워드. **간단**.
