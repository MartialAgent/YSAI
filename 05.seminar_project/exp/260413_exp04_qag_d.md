## How QAG Works

```
Task prompt → GPT generates 3 quiz questions + expected answers
Worker response → GPT answers each question from the response alone
Result: D = 1 - (correct / total)   →   D=0.000 = perfect, D=1.000 = total failure
```

**vs. baseline**: Baseline D is a single subjective score. QAG gives an auditable per-question breakdown.

## ✅ What Worked

- **E_ChurnFormula missing variable detected (D=0.333)**: QAG correctly generates a question about all 4 required variables and the `support_tier_weight` answer fails. This is the intended omission-detection working exactly as designed.
- **C_AdCopy D=0.333 (baseline was ~0.037)**: QAG catches that the ad copy doesn't always cite the exact ROI percentage. A 9× improvement in sensitivity over the baseline critic for this specific constraint.
- **Low, stable A-Scores**: Max A-Score is 0.062 — much more controlled than Exp03 (0.204). QAG produces cleaner deviation signals.
- **Auditable per-question breakdown**: Unlike holistic critic scores, you can see exactly which question failed and why.

## ❌ What's Lacking

- **QAG blind spot on D_ConditionAudit**: The quiz asks about conditions 1–4 (which were audited) and all pass. It does not ask "was condition 5 checked?" because the quiz was generated from the task prompt, not from a known ground-truth checklist. **QAG validates what was said, not what was omitted.**
- **F_BoardReport passes 3/3 despite incomplete upstream**: F references data from E which is missing a variable, yet F's quiz passes. QAG doesn't propagate knowledge of upstream omissions.
- **Quiz quality depends on GPT's question generation**: If GPT doesn't generate the "right" questions, failures are missed. This is a structural weakness — the quizzer and worker are the same model.

## 🔧 What's Needed Next

- Provide a **ground-truth checklist** to QAG so it always probes for every required condition (not just what it guesses to ask)
- Combine QAG with logprobs C for both D and C improvements (Exp03 + Exp04)
- Use G-Eval's CoT rubric for more systematic condition coverage (→ Exp05)
