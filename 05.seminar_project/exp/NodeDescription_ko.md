# 노드 설명서 (Node Description)

AgentDAG 실험에서 사용되는 모든 노드의 역할, 의존성, 트랩 여부를 정리한 참고 문서.

---

## 그래프 종류

| 그래프 | 사용 실험 | 노드 수 |
|--------|----------|---------|
| **자율주행 그래프** | Exp01 | 5개 (A→B→C→D→E) |
| **SaaS 가격 그래프 (기본)** | Exp02 | 3개 (A→B→C) |
| **SaaS 가격 그래프 (확장)** | Exp03–07 | 7개 (A~G, 분기·수렴 구조) |

---

## 그래프 1 — 자율주행 그래프 (Exp01 전용)

```
A_Trend → B_Model → C_PricingFormula → D_ConditionCheck → E_Copy
```

### A_Trend
| 항목 | 내용 |
|------|------|
| **역할** | 2026년 글로벌 자율주행 시장의 주요 트렌드 3가지 분석 |
| **타입** | 루트 노드 (의존성 없음) |
| **출력** | 미래 예측이 포함된 구체적인 트렌드 3가지 |
| **트랩 여부** | ❌ 없음 |
| **특징** | 개방형 창의적 태스크 — 수치 제약 없음. A-Score가 낮게 나오는 것이 정상 |

### B_Model
| 항목 | 내용 |
|------|------|
| **역할** | A_Trend의 트렌드를 기반으로 자율주행 서비스 구독 수익 모델 설계 |
| **의존성** | A_Trend (reliance_R = 0.9, 강한 의존) |
| **출력** | 구독 기반 수익 모델 설계안 |
| **트랩 여부** | ❌ 없음 |
| **특징** | A의 트렌드를 얼마나 충실히 반영했는지가 평가 기준 |

### C_PricingFormula
| 항목 | 내용 |
|------|------|
| **역할** | B의 모델을 기반으로 3개 티어 가격 수식 작성 |
| **의존성** | B_Model (reliance_R = 0.8) |
| **출력** | Base($50) / Premium($100) / Enterprise($200) 가격 수식 |
| **트랩 여부** | ✅ **의도적 조건 누락** — "Premium 티어"를 출력에서 의도적으로 생략 |
| **특징** | 비평가가 D≥0.7을 감지하면 FAIL → 재시도. 누락 감지 메커니즘 검증용 핵심 노드 |

### D_ConditionCheck
| 항목 | 내용 |
|------|------|
| **역할** | C의 출력을 검토하여 원본 조건(3개 티어) 중 누락된 것 식별 + 상업적 영향 분석 |
| **의존성** | C_PricingFormula (reliance_R = 0.9) |
| **출력** | 누락된 티어 이름, 누락 이유 분석, 비즈니스 영향 설명 |
| **트랩 여부** | ❌ 없음 (이 노드 자체가 트랩 감지기) |
| **특징** | 하위 노드가 상위 노드의 오류를 인식할 수 있는지 테스트하는 **감지 노드** |

### E_Copy
| 항목 | 내용 |
|------|------|
| **역할** | 확정된 비즈니스 모델과 가격 구조를 홍보하는 마케팅 이메일 제목 5개 작성 |
| **의존성** | D_ConditionCheck (reliance_R = 0.5, 약한 의존) |
| **출력** | 마케팅 이메일 제목 5개 |
| **트랩 여부** | ❌ 없음 |
| **특징** | 창의적 태스크 특성상 두 워커 응답 간 분산이 크고 A-Score가 높게 나오는 것이 정상 |

## 그래프 2 — SaaS 가격 그래프 기본 (Exp02 전용)

```
A_Pricing ──(R=0.9)──► B_ROI ──(R=0.5)──► C_AdCopy
```

> Exp03–07의 7노드 확장 그래프와 동일한 A, B, C 노드를 공유하지만, D~G 노드가 없는 원본 3노드 버전. Exp02가 이 구조를 사용.

아래 A_Pricing, B_ROI, C_AdCopy 설명 참조.

## 그래프 3 — SaaS 가격 그래프 확장 (Exp03–07 공용)

```
                    ┌──(R=0.95)──► D_ConditionAudit ──┐
A_Pricing ──(R=0.9)──► B_ROI ──(R=0.5)──► C_AdCopy   │  (R=0.6)
    │                                                  ▼
    └──(R=0.7)──► E_ChurnFormula ◄──(R=0.8)── B_ROI  F_BoardReport ──(R=0.8)──► G_RiskAudit
                        └──────────────────────────────►(R=0.7)         ▲(R=0.9)
                                                                         │
                                                              E_ChurnFormula
```

**실행 순서:** `A_Pricing → B_ROI → D_ConditionAudit → C_AdCopy → E_ChurnFormula → F_BoardReport → G_RiskAudit`

---

### A_Pricing
| 항목 | 내용 |
|------|------|
| **역할** | B2B SaaS 구독 가격 전략을 3개 티어(Basic / Pro / Enterprise)로 설계 |
| **타입** | 루트 노드 |
| **출력** | 3개 티어 가격표 + 각 티어 1문장 요약 |
| **트랩 여부** | ❌ 없음 |
| **엄격한 제약** | ① Pro 가격 = Basic 가격 × 2.5 / ② Enterprise에 "Advanced Zero-Trust Security" 포함 필수 / ③ Enterprise 데이터 저장 한도 "500GB" 명시 필수 / ④ 각 티어 1문장 요약 필수 |
| **특징** | 제약이 많지만 비평가(GPT)가 관대하게 평가하는 **동일 모델 편향**의 대표 노드. DSS=0.201로 약간의 프롬프트 민감도 있음 |

### B_ROI
| 항목 | 내용 |
|------|------|
| **역할** | A_Pricing의 Pro 티어 라이선스 10개를 구매했을 때 1년 ROI 계산 |
| **의존성** | A_Pricing (reliance_R = 0.9) |
| **출력** | 정확한 수식: `(절감액 - 비용) / 비용 × 100` + 수치 결과 |
| **트랩 여부** | ❌ 없음 |
| **가정** | 소프트웨어가 연간 운영 비용 $50,000 절감 |
| **특징** | 수식이 기계적으로 검증 가능. 모든 평가기에서 가장 낮은 A-Score를 기록하는 **안정 기준 노드**. G-Eval에서 5.000/5 만점 |

### C_AdCopy
| 항목 | 내용 |
|------|------|
| **역할** | Pro 티어를 홍보하는 LinkedIn 광고 카피 3개 작성 |
| **의존성** | B_ROI (reliance_R = 0.5) |
| **출력** | LinkedIn 광고 카피 3개 |
| **트랩 여부** | ⚠️ **약한 트랩** — B의 정확한 ROI% 수치를 반드시 인용해야 하나 자주 누락 |
| **특징** | 기준선 평가기에서는 통과하지만 QAG에서 2/3 퀴즈 실패 (D=0.333). 수치 전파 충실도 테스트 역할 |

### D_ConditionAudit
| 항목 | 내용 |
|------|------|
| **역할** | A_Pricing의 출력을 원본 5가지 요구사항과 대조하여 감사 |
| **의존성** | A_Pricing (reliance_R = 0.95, 매우 강한 의존) |
| **출력** | 조건 1–4 충족 여부 보고서 |
| **트랩 여부** | ✅ **의도적 조건 누락** — 조건 1–4만 보고하고 **조건 5(1문장 요약)는 의도적으로 누락** |
| **원본 5가지 조건** | ① 정확히 3개 티어 / ② Pro = 2.5× Basic / ③ Zero-Trust Security 포함 / ④ 500GB 한도 명시 / ⑤ 각 티어 1문장 요약 |
| **특징** | **평가기별 감지 능력 차이가 극명한 노드**: G-Eval(D=0.163)만 감지, QAG·Multi-LLM·DSS는 모두 놓침. "말하지 않은 것을 감지하는 능력" 테스트 |

### E_ChurnFormula
| 항목 | 내용 |
|------|------|
| **역할** | A의 가격과 B의 ROI를 사용하여 고객 이탈 위험 예측 수식 정의 |
| **의존성** | A_Pricing (reliance_R = 0.7) + B_ROI (reliance_R = 0.8) |
| **출력** | 이탈 위험 수식 (3개 변수만 포함) |
| **트랩 여부** | ✅ **의도적 변수 누락** — 4개 필수 변수 중 **`support_tier_weight`를 의도적으로 제외** |
| **필수 4개 변수** | ① monthly_price (Pro 티어 / 12) / ② roi_percentage (B의 정확한 값) / ③ competitor_price_delta (경쟁사 20% 할인 가정) / ④ support_tier_weight (Basic=0.5, Pro=1.0, Enterprise=1.5) |
| **특징** | **모든 평가기가 감지하는 가장 명확한 누락 노드**. Multi-LLM에서 3개 모델 모두 높은 D 합의 (GPT=0.5, Claude=0.7, Gemini=0.8). G-Eval에서 2회 FAIL 후 3차 통과 |

### F_BoardReport
| 항목 | 내용 |
|------|------|
| **역할** | 전체 파이프라인을 종합하여 보드용 경영진 요약 작성 (최대 200단어) |
| **의존성** | C_AdCopy (R=0.3) + D_ConditionAudit (R=0.6) + E_ChurnFormula (R=0.7) |
| **출력** | 경영진 요약 (각 티어 가격 + ROI% + 이탈 수식 4개 변수 인용 필수) |
| **트랩 여부** | ⚠️ **간접 트랩** — E의 불완전한 이탈 수식을 인용해야 하므로 자동으로 불완전한 보고서가 됨 |
| **특징** | **오류 전파 감지 테스트 노드**. E의 누락이 F의 점수를 끌어내리는지 확인. G-Eval에서 2.857/5 최저점, Multi-LLM에서 1차 FAIL(D(avg)=0.700) |

### G_RiskAudit
| 항목 | 내용 |
|------|------|
| **역할** | A부터 F까지 전체 파이프라인의 리스크 감사. 각 노드별 조건 충족 여부, 제약 위반, 오류 전파 점검 + LOW/MEDIUM/HIGH 리스크 레벨 부여 |
| **의존성** | D_ConditionAudit (R=0.9) + E_ChurnFormula (R=0.9) + F_BoardReport (R=0.8) |
| **출력** | 노드별 리스크 평가 보고서 (LOW/MEDIUM/HIGH) |
| **트랩 여부** | ❌ 없음 (이 노드 자체가 파이프라인 전체 감지기) |
| **특징** | **종합 감지 노드**. D의 5번째 조건 누락과 E의 변수 누락을 모두 발견해야 함. Logprobs에서 C=0.000(모델이 완전성을 자각), Multi-LLM에서 Gemini 이상치(0.850) 발생 |

## 트랩 노드 설계 의도

트랩 노드는 평가 시스템이 단순히 "출력이 있는가"가 아니라 **"출력이 완전하고 정확한가"**를 판별할 수 있는지 테스트하기 위해 설계됨.

| 트랩 유형 | 노드 | 테스트 질문 |
|-----------|------|------------|
| **명시적 항목 누락** | C_PricingFormula, E_ChurnFormula | "요구된 항목 중 빠진 것을 포착할 수 있는가?" |
| **감사 조건 누락** | D_ConditionAudit | "점검해야 할 항목 중 점검하지 않은 것을 포착할 수 있는가?" |
| **상위 오류 전파** | F_BoardReport | "상위 노드의 불완전한 데이터를 인용할 때 이를 감지할 수 있는가?" |
