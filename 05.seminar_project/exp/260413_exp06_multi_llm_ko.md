## ✅ 잘 된 점

- **E_ChurnFormula: 3개 판정 모두 높은 D**: GPT=0.500, Claude=0.700, Gemini=0.800. 서로 다른 모델 계열 3개가 독립적으로 높은 편차에 동의하면 위양성 확률이 매우 낮음. 7개 실험 전체에서 **가장 신뢰할 수 있는 누락 감지**.
- **F_BoardReport FAIL 트리거 (D(avg)=0.700)**: E의 오류 전파 감지 — 불완전한 이탈 데이터 인용을 모든 판정이 벌점. FAIL→재시도 루프가 정상 복구.
- **동일 모델 편향 보정**: 기준선에서 GPT 전용 D(A_Pricing)는 ~0.05. 여기서 GPT-4o 단독은 0.500 (더 진실에 가까움). Claude/Gemini 평균으로 0.167로 중화됨.
- **B_ROI·C_AdCopy 만장일치 D=0.000**: 교차 모델 합의가 깨끗한 노드에서 강력한 품질 신호 — 3개 모두 괜찮다고 하면 진짜 괜찮음.

## ❌ 부족한 점

- **D_ConditionAudit: 3개 판정 모두 D=0.000**: QAG처럼 멀티 LLM도 감사 보고서에서 누락된 5번째 조건을 감지하지 못함. 판정이 있는 것을 평가하지, 없는 것을 평가하지 않음. G-Eval을 제외한 모든 평가기의 일관된 맹점.
- **G_RiskAudit에서 Gemini 이상치 (0.850 vs 0.200)**: Gemini와 나머지 간 0.65 차이는 의미 있게 평균하기에 너무 큼. 파이프라인 리스크 평가에서 Gemini가 모델 특정 엄격 기준 적용. 평균(0.417)이 올바른 신호가 아닐 수 있음.
- **비용이 기준선의 3배**: 노드당 3번 판정 호출. 7노드 그래프에서 실행당 추가 API 호출 21회.
