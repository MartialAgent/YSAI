## 1. 한 줄 요약

MAS(다중 에이전트 시스템)의 통신을 **signed DAG**로 모델링한 뒤, 최종 답변으로부터 **PageRank 스타일 역전파(backward propagation)**로 각 에이전트의 기여도를 산출해 **악성 에이전트를 학습 없이 식별·격리**하는 동적 방어 기법.

---

## 2. 문제 정의 (Why)

### 2.1 MAS의 취약성
- LLM 기반 MAS는 소프트웨어 엔지니어링, 시장 분석, 웹 작업 등에서 빠르게 확산.
- 단일 LLM과 달리 MAS는 **전염성(contagious)** 공격에 노출됨: 한 에이전트의 출력을 조작하면 인접 에이전트로 전파되어 시스템 전체가 붕괴.
- 본 논문은 이를 **corruption attacks against MAS**로 정의.

### 2.2 기존 방어의 한계
| 방어 계열 | 대표 기법 | 한계 |
|---|---|---|
| Output-based | BlockAgents, AgentForest | 미세한 텍스트 변조에 취약, 평가자가 공격당하면 무력화 |
| Static graph 기반 | Huang et al. (2025) | 고정된 토폴로지에만 최적화, 동적 환경에 적응 불가 |
| 학습된 분류기 | G-Safeguard | **국소 신호**만 사용 → 최종 결정으로의 전파 양상을 보지 못함, 토폴로지 변경 시 재학습 필요 |

→ **글로벌 관점에서, 학습 없이, 동적 토폴로지에 적응하는 방어가 필요**.

---

## 3. 방법론: BPD (Backward Propagation Detection)

### 3.1 MAS → Signed Temporal DAG로 재구성
- 에이전트 집합 A = {A(1), ..., A(n)}, 총 T 라운드 통신.
- 각 라운드 t에서 에이전트 A(i)를 **temporal node** A_t(i)로 분해.
- 엣지 e_t(i, j): 라운드 t의 A_t(i)에서 t+1의 A_{t+1}(j)로 전달된 메시지.
- 시간이 단조 증가하므로 사이클이 생기지 않음 → 자동으로 **DAG**.

### 3.2 Signed Edge Scoring
- 각 엣지 e_ij에 대해 외부 LLM 평가자 f가 신호 g_ij ∈ {-1, 0, +1}을 부여:
  - `+1`: 수신자가 송신자에게 동의/신뢰
  - ` 0`: 무시 / 기여 없음
  - `-1`: 반박 / 불신
- 평가용 LLM은 MAS와 **독립**적으로 동작 (Appendix A.3에 프롬프트).

### 3.3 Node Contribution via Backward Propagation
1. **Signed Influence Operator**
   B = D⁻¹ G  (G는 부호화된 인접행렬, D는 out-degree 대각행렬)

2. **Boundary Condition (최종 답변에서 시작)**
   - S(A_T(i)) = +1, A(i)의 답이 MAS의 최종 답과 일치할 때
   - S(A_T(i)) = -1, 그렇지 않을 때

3. **Backward Pass (PageRank 스타일)**
   - S(C_i) = (1/k_i) Σ_{j ∈ N⁺(i)} g_ij · S(C_j)
   - DAG이므로 **단일 토폴로지 패스**로 수렴 (반복 불필요).
   - 일반적인 PageRank의 damping/teleportation을 boundary 초기화로 대체.

4. **Agent-level Aggregation**
   - Ŝ(A(i)) = 라운드 평균
   - 편차 δ(i) = (1/(n-1)) Σ_{j≠i} |Ŝ(A(i)) - Ŝ(A(j))|

5. **Outlier Detection & Mitigation**
   - M = {A(i) | δ(i) ≥ τ}   (논문에서 τ = 1.5)
   - 검출된 에이전트의 **모든 송신 엣지를 제거**하여 그래프 복구.

### 3.4 핵심 설계 특징
- **No Training**: 각 질의마다 즉석에서 detection/repair → 새로운 토폴로지에 즉시 적응.
- **Efficient**: DAG 위 단일 backward pass.
- **Interpretable**: 각 노드의 점수가 최종 결정에 대한 부호화된 누적 영향력으로 직접 해석됨.

## 4. 실험 설계

### 4.1 환경
- **데이터셋**
  - 지식 QA: MMLU (Algebra, Math, Chemistry, Computer, Security 각 100문항)
  - 개방형 생성: Alpaca, Samsum, ChatDoctor
- **LLM 백본**: GPT-4o, DeepSeek-V3
- **MAS 구조**
  - **Flat**: 5개 에이전트 동등 협업
  - **Hierarchy**: 5 응답자 + 2 평가자 → 응답자 수정
- **평가 지표**
  - Answer ACC (작업 정확도)
  - Monitor ACC (악성 에이전트 식별 정확도)
  - BLEURT, GPT-4 점수 (개방형 생성)
- **공격 5종**: Harmful, Suboptimal, Reframing, Trigger, Modification
- **베이스라인**: G-Safeguard, AGENTXPOSED, Challenger, Inspector

## 5. 주요 결과

### 5.1 고정 그래프 (MMLU, GPT-4o)
| 구조 | 공격 시 | G-Safeguard | **BPD** |
|---|---|---|---|
| Flat | 76.3% | 83.7% | **85.4%** |
| Hierarchy | 78.6% | 85.5% | **87.5%** |

- 공격 하 Answer ACC를 **+10pp / +8pp** 회복.
- Monitor ACC: G-Safeguard 대비 Flat **+2pp**, Hierarchy **+6pp** (학습 없이!).

### 5.2 개방형 생성 (Alpaca/Samsum/ChatDoctor)
- 공격 시 BLEURT는 15.7~15.8% 감소, GPT 점수는 8.8~11.9% 감소.
- **BPD 적용 후 감소폭이 BLEURT 1.3~1.5%, GPT 0.6~0.8%로 축소**.
- Monitor ACC 평균 **95%**.

### 5.3 공격 유형별 강건성 (Table 5)
- 평균: BPD가 모든 베이스라인 대비 **+6~7pp**.
- **Modification 공격(미세 의미 변조)에서 +10~16pp** 큰 격차 → 텍스트 유사도 기반 검출이 놓치는 미묘한 공격을 BPD가 잡아냄.

### 5.4 동적 그래프
- 그래프 토폴로지와 공격 대상이 시간에 따라 변화.
- 베이스라인은 정적 환경 대비 평균 **3% 성능 저하**.
- BPD는 정적 시 성능을 거의 그대로 유지 (Flat 88%, Hierarchy 85%).

### 5.5 Ablation: Backpropagation의 필요성
- BP 없이 원시 점수만으로 검출 → Response ACC 2~3%↓, Monitor ACC 6~8%↓.
- → **국소 distrust 신호만으로는 부족**, 글로벌 전파를 통한 누적 영향력이 핵심.

### 5.6 Threshold τ 민감도
- τ = 1.3 → detection rate 96%지만 정상 엣지 제거 → ACC 저하
- τ = 1.7 → detection rate 47%로 떨어져 공격 잔존
- **τ = 1.5가 최적** (평균 ACC 87%).

### 5.7 Rater 공격 / System-Aware 적응 공격
- 스코어링 자체를 노린 적응적 공격 하에도 3~4% 정도의 미미한 저하만 발생.
- BPD는 다수와의 편차로 판별하므로 일부 점수 왜곡에 강건.
- SA 공격 하에도 평균 ACC 83~85% 유지, 일반 베이스라인의 **무공격 성능보다 우월**.

### 5.8 결합 공격
- Harmful+Reframing, Suboptimal+Reframing 모두 정상 대비 2~3%만 저하.
- 베이스라인은 큰 폭의 성능 하락.

### 5.9 계산 비용 (Table 15)
- 표준 MAS 대비 추가 시간 오버헤드:
  - Flat: **+9.0%**
  - Hierarchy: **+10.6%**
- 외부 평가자가 문장쌍만 채점하므로 토큰 비용 미미.

## 6. 핵심 기여 (저자 주장)
1. MAS 통신을 **signed DAG**로 정식화하여 에이전트 영향력의 **글로벌 분석** 가능.
2. **Backward propagation 기반 검출** 알고리즘 제안 — 학습 없이 노드 기여도 계산.
3. 다양한 MAS 구조/공격/모델 조합에서 일관된 성능 우위, 특히 **동적·미세변조 시나리오에서 큰 격차**.

## 7. 한계와 향후 과제
- **Byzantine 한계 초과 공격** 및 스코어링 외부를 노리는 공격은 본 연구 범위 밖.

---

## 8. 학습 노트 — 왜 흥미로운가
- **PageRank의 부호 확장판을 DAG에 적용**한 발상이 깔끔. 반복 수렴 없이 단일 backward pass로 글로벌 영향력을 산출하는 점이 효율성의 핵심.
- 기존 GNN 기반 방어가 풀지 못한 "전파 양상(propagation pattern)"을 **boundary condition을 최종 답변에 두고 거꾸로 전파**하는 방식으로 우회.
- **학습이 필요 없다**는 점이 동적 MAS에 대한 가장 큰 실용적 우위 — 토폴로지/공격 분포가 바뀔 때마다 분류기를 재학습하지 않아도 됨.
- 본인 연구(**AgentDAG**)와의 접점: signed DAG 모델링, 노드 기여도 분석, 동적 그래프 강건성 — 비교/차별화 지점을 명확히 둘 가치 있음.
