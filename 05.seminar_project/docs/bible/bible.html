DAG 기술 기초

DAG란

Directed Acyclic Graph — 노드(작업 단위) + 방향 엣지(데이터 의존성) + 비순환 보장.
실행 순서는 위상 정렬(Topological Sort)로 결정론적으로 결정.

Linear:    A → B → C
Parallel:  A → B, A → C      (B·C 동시)
Fan-in:    B → D, C → D      (D는 B·C 완료 후)
Diamond:   A→B→D, A→C→D     (Fan-out+Fan-in)
7대 표준정의
위상 정렬선행 노드 항상 먼저 실행
Critical Path전체 실행 시간 결정 최장 경로
Dataflow 모델데이터 가용성이 실행 트리거
멱등성동일 입력 → 동일 출력 (재시도 안전)
체크포인팅노드 완료 시 상태 영속 저장
Fan-out/Fan-in분배 → 합산 패턴
비순환 보장정의 시점 DFS로 사이클 차단
구성 요소역할
Orchestrator위상 정렬 → 실행 순서 결정, 노드 상태 추적, 재시도·HITL 라우팅
Worker개별 노드 작업 수행 (LLM 호출)
State Store노드별 입출력·상태 영속 저장
Evaluator노드 출력 품질 판정 → PASS / FAIL / AMBIGUOUS

EPR 선행 연구 — AgentEval & AgentProp-Bench

AgentEval (arXiv:2604.23581)

에이전트 실행을 평가 DAG로 형식화, 노드별 LLM Judge 부착

  • 루트코즈 실패 36.8% / 전파 실패 63.2%
  • 평균 전파 체인 길이: 2.1 노드
  • FDRec 0.89 vs 0.41 (E2E 대비 2.17×)
  • 인간 전문가 합의 κ = 0.84
→ 루트코즈 vs 전파 실패 구분으로 게이트 효과 직접 측정 가능

AgentProp-Bench (arXiv:2604.16706)

파라미터 수준 fault injection 후 3단계 지표로 전파율 분해

S1 = 주입 적용됨
S2 = 하위 실행에 observable 변화
S3 = 최종 출력 오류

EPR = P[S3|S1] ≈ 0.62
      (범위: 0.46 ~ 0.73)
  • 거절률·회복률은 독립 능력 (ρ=0.126)
  • 전파율 모델별 편차 큼 (0.43 ~ 0.97)
→ S1→S3 추적으로 게이트 개입 단계 정량화

EPR 선행 연구 — MAS-FIRE & Sherlock

MAS-FIRE (arXiv:2602.19843)

15개 결함 유형 × 3가지 비침습 주입 방식으로 MAS 강건성 평가

Of = 결함 인식 후 방어 행동 유발 비율
Lf = 지역 복구 성공률
Sf = 지역 복구 → 전역 성공 비율
Lf - Sf 갭 = 잔류 전파 효과
  • 반복 폐루프: 선형 대비 결함 40%+ 무력화
  • 아키텍처 토폴로지 ≥ 모델 크기
→ Option2 백프롭의 Lf/Sf 차이로 보완 측정

Sherlock (arXiv:2511.00330)

반사실적 분석으로 취약 노드 식별 → 선택적 검증기 부착

노드 취약성 순서:
말단 노드 > 초기 노드 > 중간 노드

말단: 복구 경로 없음
초기: 오류가 전체 파급
중간: 하위 노드가 보정 가능
  • Fan-in ↑ → 오류 증폭 가능성 ↑
  • 정확도 +18.3% / 실행시간 -48.7%
→ A_Pricing(초기 노드) 주입 위치 이론적 뒷받침
