# PRD: 클로드 코드 기반 로컬 무한 학습 시스템

## 1. 개요

### 제품명
**AI Study Loop** — Claude Code와 함께하는 로컬 무한 학습 시스템

### 목적
사용자가 AI/ML 핵심 지식을 체계적으로 학습할 수 있도록, Claude Code가 로컬 학습 자료(MD 파일)를 기반으로 대화·출제·피드백을 무한 반복하는 자율 학습 루프를 구현한다.

### 배경
- 이론서는 있지만 "대화하며 확인받는" 학습 파트너가 없다.
- 온라인 강의는 단방향이며, 진도 맞춤 피드백이 없다.
- Claude Code는 로컬 파일을 읽고 스크립트를 실행하며 대화할 수 있어, 개인화 학습 루프 구현에 최적이다.

---

## 2. 학습 도메인 (6개 모듈)

각 모듈은 독립된 MD 파일로 관리된다.

| 파일명 | 도메인 | 핵심 내용 |
|---|---|---|
| `prob_stats.md` | 확률통계 | 확률분포, 기댓값, 베이즈 정리, MLE, 정보이론 |
| `linear_algebra.md` | 선형대수 | 벡터·행렬 연산, 고유값, SVD, 차원축소 |
| `python_syntax.md` | 파이썬 문법 | 자료구조, 함수형, OOP, NumPy/Pandas 패턴 |
| `nlp.md` | NLP 개념과 코드 | 토크나이저, 임베딩, Attention, 전처리 파이프라인 |
| `llm.md` | LLM 개념과 코드 | Transformer 구조, 파인튜닝, RAG, 프롬프트 엔지니어링 |
| `agent.md` | 에이전트 개념과 코드 | ReAct, Tool Use, 메모리, 멀티에이전트 오케스트레이션 |

## 3. 시스템 아키텍처

```
[로컬 파일 시스템]
  ├── prob_stats.md
  ├── linear_algebra.md
  ├── python_syntax.md
  ├── nlp.md
  ├── llm.md
  ├── agent.md
  ├── progress.json        ← 학습 진도 및 오답 기록
  └── session_log.md       ← 세션별 대화 로그

[Claude Code CLI]
  └── /loop 스킬로 무한 루프 실행
        ↓
  [학습 루프 사이클]
  1. MD 파일 읽기 (해당 모듈)
  2. 개념 설명 or 문제 출제
  3. 사용자 답변 수신
  4. 코드 실행·검증 (python 스크립트)
  5. 피드백 제공
  6. progress.json 업데이트
  7. 다음 토픽으로 이동 또는 반복
```

## 4. 기능 요구사항

### 4.1 학습 루프 (핵심)

| ID | 기능 | 설명 |
|---|---|---|
| F-01 | 모듈 선택 | 사용자가 세션 시작 시 학습할 모듈 선택 |
| F-02 | 개념 설명 | MD 파일의 해당 섹션을 읽어 대화체로 설명 |
| F-03 | 문제 출제 | 객관식·주관식·코딩 문제를 자동 생성 |
| F-04 | 코드 실행 | 사용자가 작성한 Python 코드를 로컬에서 실행·검증 |
| F-05 | 즉시 피드백 | 오답 원인 분석 + 올바른 개념 재설명 |
| F-06 | 진도 추적 | `progress.json`에 맞춤·오답·완료 토픽 기록 |
| F-07 | 약점 반복 | 오답률 높은 토픽을 다음 세션에 우선 재출제 |
| F-08 | 무한 루프 | `/loop` 스킬로 세션 종료 없이 학습 지속 |

### 4.2 모듈별 학습 방식

**확률통계 / 선형대수**
- 수식 설명 → NumPy로 직접 계산 → 결과 검증
- 예: "정규분포에서 σ=2일 때 P(X<1)을 scipy로 계산하라"

**파이썬 문법**
- 코드 스니펫 제시 → 동작 예측 → 실행으로 확인
- 예: `list(map(lambda x: x**2, range(5)))` 출력값은?

**NLP**
- 개념 설명 → HuggingFace/NLTK 코드 실습
- 예: BPE 토크나이저를 직접 구현하기

**LLM**
- 아키텍처 설명 → Anthropic API 또는 로컬 모델로 실습
- 예: "temperature 0.0 vs 1.0 차이를 실제 출력으로 비교하라"

**에이전트**
- ReAct 루프 개념 → Tool Use 코드 작성 → 멀티에이전트 구성
- 예: 도구를 가진 Claude Code 에이전트를 직접 구현

### 4.3 세션 관리

- `progress.json` 스키마:
  ```json
  {
    "module": "nlp",
    "completed_topics": ["tokenization", "embedding"],
    "weak_topics": ["attention_mechanism"],
    "total_questions": 42,
    "correct": 35,
    "last_session": "2026-05-11"
  }
  ```
- 세션 재개 시 이전 진도에서 이어서 시작

## 5. 비기능 요구사항

| 항목 | 요구사항 |
|---|---|
| 실행 환경 | Windows 11 로컬, Python 3.10+, Claude Code CLI |
| 외부 의존성 | 없음 (인터넷 없이도 MD 파일 기반 학습 가능, API 실습 시 선택적 연결) |
| 응답 속도 | 문제 출제~피드백 사이클 30초 이내 |
| 확장성 | 새 MD 파일 추가만으로 새 모듈 편입 가능 |
| 데이터 보존 | `progress.json`, `session_log.md` 로컬 저장 |

## 6. 사용자 플로우

```
1. 터미널에서 Claude Code 실행
   > claude

2. 학습 루프 시작
   > /loop AI 학습 시스템 — prob_stats.md부터 시작

3. Claude Code가 루프 진입:
   - progress.json 읽기
   - 오늘의 토픽 선정
   - 개념 설명 출력
   - 문제 출제
   - 사용자 답변 대기

4. 사용자 답변 입력 (텍스트 or 코드)

5. Claude Code가:
   - 코드면 → 실행 후 결과 비교
   - 개념이면 → 정오 판단 + 피드백

6. progress.json 업데이트

7. 다음 토픽으로 이동 → 3번으로 반복
```

## 7. 구현 마일스톤

| 단계 | 작업 | 산출물 |
|---|---|---|
| M1 | 6개 MD 학습 파일 초안 작성 | `prob_stats.md` 외 5개 |
| M2 | `progress.json` 스키마 및 초기화 스크립트 | `init_progress.py` |

| M4 | 모듈별 학습 루프 테스트 (각 1시간) | 테스트 로그 |
| M5 | 약점 반복 알고리즘 검증 | `progress.json` 누적 데이터 |
